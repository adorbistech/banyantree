# BanyanTree — Project Context

## The one sentence

> BanyanTree remembers WHY your code is the way it is — not just what it is.

---

## The problem

Software projects lose memory faster than humans can rebuild it.

Every large codebase eventually suffers from:
- Forgotten architectural decisions
- Lost reasoning behind design choices
- Tribal knowledge that walks out the door
- Context switching fatigue
- AI tools that start every session blank

Current AI tools understand **what the code says right now**.
They do not preserve **why systems were built**, **how workflows evolved**, or **what constraints shaped the architecture**.

**Code survives. Reasoning dies.**

---

## The aha moment — the product in one scenario

A developer opens a file they haven't touched in 6 weeks.

Without typing a single word, the sidebar already shows:

- *"Last worked on: 6 weeks ago"*
- *"Open question from that session: why does the retry logic not handle 429s?"*
- *"Auth root connected — token expiry rule: max 24h"*
- *"2 related branches from the auth refactor 3 weeks ago"*

They didn't ask. It was already there.

That 10-second experience — the feeling that **the project remembers itself** — is the product.

Everything BanyanTree builds is in service of that single moment.

---

## What BanyanTree is

**Category:** Local-first repository cognition runtime

**Not:** AI coding assistant, code generator, SaaS platform, autonomous agent, productivity tracker

It is infrastructure. Like Git — but for cognitive continuity instead of version history.

---

## What Phase 1 proves

One developer. One repository. One sidebar. One graph. One memory stream.

Phase 1 succeeds if a developer can return to an old repository after 1+ weeks away and recover full working context in under 3 minutes — without manually re-reading the codebase.

That is the entire success criterion.

---

## The core principle

> Memory is shown to the developer first. Claude receives it second.

BanyanTree amplifies human reasoning. It does not replace it.
The developer sees context, understands it, decides what matters — then asks Claude.
The human is always in the loop before the AI is.

---

## Why the name

A banyan tree grows interconnected roots, branches, and support systems — expanding organically over time, with old roots feeding new growth.

That maps to BanyanTree exactly:
- Graph cognition = interconnected roots
- Memory relationships = branches
- Architectural evolution = living expansion over time
- Persistent knowledge networks = the whole living structure

The repository becomes a cognitive structure — not static files.
